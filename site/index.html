THE PROBLEM

Bitcoin is not quantum-safe.

ECDSA (secp256k1) — the signature scheme securing every Bitcoin transaction — is broken by Shor's algorithm on a quantum computer. This isn't hypothetical. It's an inevitability.

🔓

Private Key Extraction

Shor's algorithm can derive a private key from a public key in polynomial time on a quantum computer. Every address with a revealed public key is vulnerable.

💰

~4M BTC at Risk

Approximately 4 million Bitcoin sit on addresses with exposed public keys. A quantum adversary could steal all of them in minutes.

⏳

No Migration Plan

Bitcoin has no roadmap for post-quantum migration. A hard fork would be required, and backward compatibility is essentially impossible.

THE SOLUTION

How BitcoinQS works

Three fundamental changes make BitcoinQS quantum-secure.

01

WOTS+ Signatures

Replace ECDSA with WOTS+ — a hash-based one-time signature scheme. Security relies solely on SHA-256 preimage resistance, which remains strong against quantum attacks (Grover's only halves bit security).

  • 67 hash chains per keypair
  • No algebraic structure to exploit
  • NIST-aligned (XMSS building block)
02

One-Time Key Rotation

Each WOTS+ key signs exactly one transaction, then rotates. This isn't a limitation — it's forward secrecy by design.

  • Compromised key → only 1 tx affected
  • Change goes to fresh keys by default
  • Automatic wallet key management
03

Hybrid Addressing

Addresses derived from both SHA-256 and SHA3-256 of the public key root, then combined. Double-layered hash security.

  • SHA-256 + SHA3-256 dual hashing
  • bqs1... prefix for network identity
  • Resistant even if one hash weakens
